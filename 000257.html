自动监控主页，篡改短信通知

　　10.1长假，很多网管的一个重要任务是监控自己的主页被黑了没有。下面写了个脚本，把页面的变动用手机短信提醒一下，这样就不用定期用浏览器进行访问了。

　　本脚本在red Hat Linux 7.2上调试通过。
1、页面比较，异常则给某个信箱发email.

#/bin/sh

#

#pagecmp.sh <url> <pageid> <N|不作比较的特殊行的标识字符，如访客统计等>

#

echo -n $1

wget $1 -O $2.tmp.html -o log.candel

if [ "$3" = "N" ] ; then

        echo -n "[ no grep ]";mv $2.tmp.html $2.cur.html

else

        grep -v "$3" $2.tmp.html >$2.cur.html

fi

diff $2.cur.html $2.html >$2.diff.log;a=$?

if [ "$a" = "0" ] ; then

        echo "                          :same"

else

        echo "                          :changed"

        ( echo -n "检查时间:"; date -R ;

        echo;echo "$2  的主页被修改";

        echo "---------------------------------------------------------";

        echo;echo "diff $2.cur.html $2.html";

        echo;cat $2.diff.log) | mail yourmail@yourdomain.com -s "紧急通知：$2 的主页被修改"

fi

2、总控制脚本

#/bin/sh

#

#all.sh 

#

#	手工添加要监控的url.

#	在将url加到这儿之前，需要作初始化，以hedong为例

# 		./pagecmp.sh http://hedong.3322.org hedong N

# 		mv hedong.cur.html hedong.html

#	其中hedong.html是基准文件

#

cd /path/to/script/

./pagecmp.sh http://hedong.3322.org hedong N

./pagecmp.sh http://www.nstl.gov.cn/nstl/user/ywjsdg.jsp "全文检索订购" "网站访问次数"

#...

3、将总控脚本加入自动任务
运行

crontab -e

然后加入以下两行
#home page check
18,48 * * * * ( /path/to/script/all.sh >/dev/null 2>&1)

　　此处设为每小时的第18、48分钟进行检测，可据情况修改。

4、关于短信提醒
　　我用的是新浪的免费信箱，有短信提醒功能。建议先试试短信功能是否可行，我的就是申请后经人工服务手工启动的。
