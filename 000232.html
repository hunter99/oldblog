SAXAPI学习笔记（2）

　　没想到SAXAPI的学习还是走了弯路，直接到SAX网站读其文档和例程是最快最准因而也最好的方法。

一、几个细节
1、SAX是一个通用接口，就象JDBC提供了对各种关系型数据库的操作接口一样，SAX提供了各种XML解析器的通用接口。(在1中我还误认为JAXP是这么个角色，SAX是实现呢，虽然从更高的层面上看有一定的道理）
2、使用sax2的三个条件：java1.1或更高版本，classpath中指明sax2兼容的XML解析器，同时将sax2包放在classpath中。
3、SAX驱动器的选择，有三种方法：1）编译SAX2时指定默认的驱动，2）在命令行通过-D参数指定，如java -Dorg.xml.sax.driver=com.example.xml.SAXDriver MySAXApp sample.xml ,3）通过在程序直接指定驱动器生成XMLReader对象。
4、SAX2定义了标准的方法，来取得或设置XMLReader对象中的功能标志或属性值。通过getFeature, setFeature, getProperty, and setProperty 等方法，可以改变分析器的行为（如要求分析检验一个XML文档的合法性），或注册新的事件处理类型。
　　功能标志的类型是boolean型，而属性的类型则可为任意对象，功能和属性的名字是完整的URI。
　　功能和属性有关的异常有两个：
SAXNotRecognizedException当试图查找或设置一个并不存在的功能标志时抛出（常见多因URI写错了）
SAXNotSupportedException当试图设置一个不支持的功能的状态或那个状态只读时抛出

　　属性的一个重要应用，是取出或设置扩展的事件处理器。

5、过滤器
filter
SAX接口假定两个基础的流，一个是事件流从SAX驱动器（通常是一个XMLReader）流向应用程序，一个是请求流从应用程序流向SAX驱动器。
SAX为过滤器的实现提供一了个接口org.xml.sax.XMLFilter和一个helper类org.xml.sax.helpers.XMLFilterImpl。
XMLFilter接口本身非常简单，扩展了XMLReader如下。换言之，XMLFilter是一个以其它XMLReader为parent的XMLReader,对parent XMLReader而言它是处理事件的应用程序，对应用程序而言它又是产生events的SAX驱动器。

public interface XMLFilter extends XMLReader
{
public abstract void setParent (XMLReader parent);
public abstract XMLReader getParent ();
}

XMLFilterImpl提供了一个定义SAX2过滤器的基础，实现XMLFilter, EntityResolver, DTDHandler, ContentHandler, and ErrorHandler 等接口（DeclHandler and LexicalHandler等扩展接口没有实现）。
在具体的过滤器实现时，只需要调用super.*相关方法，就可以将流的信息继续传输。

6、命名空间
SAX2增加了对命名空间的支持，这影响了 ContentHandler和Attributes 接口，SAX1中只在qName中的信息，在SAX2用URI和localName两变量来存放。在startElment和endElement中，取得相关的名字的时，要先判断URI是否为空。在学习笔记1中，好象是判断qName了，不是正统的手法。
对Attributes，可以用getURI,getLocalName和getValue来取得相应的值。如果URI为空，要用qNamed而不是localName.
对命名空间处理过程的设置
"http://xml.org/sax/features/namespaces"功能控制基本的命名空间处理，http://xml.org/sax/features/namespace-prefixes功能控制报告 qNames 和命名空间声明 (xmlns* attributes)。
