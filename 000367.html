学习struts笔记（2）

　　昨天开了小差，今天继续学习。

学习struts


　　 关于这个框架的官方文档，可以在线获得，也可以从随struts发布的一个自安装的struts-documentation.war程序包中得到。这个struts-documentation.war包括：
  • 用户和开发指南

  • 全面的API文档

  • FAQ 和howto指南

  • 以及你在这儿找到的其它的内容

　　欢迎你先浏览一下在线文档，然后再于本地安装以就近学习。
注意: 如果你在浏览这个站点的文档，这一节的大多数链接都是指向每日创建的文档（ Nightly Build）. 当学习Struts时，一定要看与实际应用的struts版本一致的文档。在每个发布版本中，文档以一个应用程序的形式提供，你可以在本地安装和使用。

　　简明的 Struts 用户指南，介绍了 Model-View-Controller 架构，以及它如何跟struts的基本组件联系起来。如果你想搞清楚 "Struts是如何工作的", 这就是一个开始的地方。除了这个架构的概述，用户指还包括安装指令的细节，以及struts这个框架的每个版本的发布说明。
　　开发指南则有struts中提供的组件和扩展的详细技术参考。除了有 Struts Utilities 和 Struts Validator的指南外，还有每个包的概述和每个struts标签库的API参考。开发指南被设计用来做每日参考的，来帮助你最有效地使用这些标准包。到各种开发指南的链接构成了用户指南菜单的一部分。

　　要得到关于某个类或包的更多的细节，可以参考精心维护、非常完整的strutsAPI文档。强烈建议你在开始使用某个类时参考一下它的API文档，这会确保你不会忽略了它的某些重要的功能或选项。那些你不知道的，就不能为你所用。
　　Struts FAQ被设计用来补充那些API文档、用户/开发指南的微小遗漏。

  • Kickstart FAQ 回答常见的关于struts的非技术问题

  • Struts Newbie FAQ 回答常见的第一次使用struts的新手提出的技术问题

  • How to Help FAQ 回答常见的关于使用struts开发的项目的发布的问题
.

　　Howto Guides 被设计用来帮助你使用可选的struts扩展和组件，包括了一些象在struts中使用Secure Socket Layer (SSL)协议及单元测试struts应用程序等此类的题目。

　　Struts Wiki是Struts documentation中一个相对崭新的成员。所有的Struts社区的成员都可以在wiki上发表评论，也包括用户列表。Wiki非常适合持续讨论你认为重要的话题，而邮件列表则重适合于讨论突发的问题。

　　如果你想对在这儿看到的页面发表评论，你就可以据这个链接到wiki上发表。

　　要获取更多的帮助，你可以试一下JGuru FAQ 和 Forum的JGuru Struts知识库。

　　当然，最好的文档是代码本身。如果你对Struts在实际中如何工作有疑问，不要犹豫阅读源码。整个struts框架的完整的可编译的源码，在struts分发源码的src目录下。


struts的例程


　　为帮助你理解各部分是如何组合在一起工作的，发布struts有几个例程：
  • Blank - ，一个开发新的struts应用程序的简单模板

  • Exercise-Taglib -,一组测试页面，同时展示了如何应用自定义标签

  • Documentation - ,struts文档，同在网站上看到的一样

  • MailReader - ，最原始的例程，先看一下这个

  • Tiles-Doc -, Tiles扩展的补充演示

  • Upload -,演示使用文件上传功能（基于Commons Upload.)

  • Validator - ，演示使用Validator扩展
.

　　在struts的二进制发布包中，每个程序都是一个独立的WAR文件。它们的源代码，对struts二进制发布则在各的WEB-INF/src目录下，对struts源码发布则在各web目录下。.

　　还有很多第三方的例程供学习，包括：

  • AppFuse - ，演示在struts中使用XDoclet，还不同的安全包，以及实现数据持久性的Hibernate技术

  • JPetStore - ,Java Petstore应用的改进版本，用Struts 和 iBATIS实现。
  • LogWeb - ,一个Struts应用程序，在一个servlet容器内动态配置Log4J.

　　请参见资源一节的完整的免费可得的Struts例程列表。

进一步学习


　　Struts Mailing Lists 是非常有用的、交互的信息宝库。用户列表往往积累了大量的信息，因此在张帖新问题前，一定要先检查一下出版的文档和列表文档。很有可能，这个问题已经有人问过并有了回答。
　　如果你在FAQ或列表文档中实在找不到问题的答案，你可将问题张帖到Struts 用户列表上 C 在张帖前，你必须已经订阅了列表或列表摘要。.
　　注意，Struts User 流行 "轻松星期五（casual friday）". 只要在标题前加[FRIDAY]前缀，如struts无关的消息就可以发布，且为大家所接受。如果你不想为这类消息打扰，可以据此设置邮件过滤。强烈建议你不要在其它的工作日张帖 [FRIDAY] 信息。我们非常感谢你的合作。

　　Struts资源页面提供了一些其它公开资源链接，包括

  • 书,

  • 文章,

  • 教程,

  • 研讨会, 及

  • 例程.

　　Struts News & Status页面报道影响struts的近期事件和概述当前项目的进展。

　　Roadmap 页面描述了我们未来开发的计划。

　　Struts Wiki是Struts documentation中一个相对崭新的成员。欢迎所有的Struts社区的成员（包括你）在wiki上发表评论，. 然后,Wiki不是一个提交突发问题的地方。所有请求帮助的问题，要发到Struts用户列表 或其它支持论坛。

　　还有一些Wiki 页面接收对文档的评论，点击这些页面底部的 "Comments?" 链接就可去发表评论。
