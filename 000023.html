OSW:13.WebLog的分析

　　车东的文章精而不多，但WebLog分析的文章就占了两篇。看了他给BlogChina所做的分析，确实比较有价值。

　　我自己以前也做过一个，用的是webalizer-2.01_06-11,可以通过http://egroups.istic.ac.cn/wa/访问。

　　现把安装AWStats过程列在下面，供有志者参考。

　　1、去AWStats项目主页把awstats-5.6.tgz下载下来，上传到服务器，并将其解到/tmp/awstats-5.6目录下（正式安装时不要在tmp目录下啊).

cd /tmp
cp ~/awstats-5.6.tgz .
tar zxvf awstats-5.6.tgz

　　2、修改Apache配置文件了。我的是/etc/httpd/conf/httpd.conf,你的我可不知道在哪个目录下，名字应该一致吧。在Awstats的安装说明中，是将目录分放在不同的目录下，我怕麻烦，就不分目录了。设设置如下：
增加：

Alias /awstat/ "/tmp/awstats-5.6/wwwroot/"
<Directory "/tmp/awstats-5.6/wwwroot">
    Options Indexes MultiViews Includes FollowSymLinks +ExecCGI
    AllowOverride All
    Order allow,deny
    Allow from all
</Directory>

修改：
AddHandler cgi-script .cgi .pl
CustomLog logs/access_log combined

　　3、修改awstats.conf，最困难的时刻来了。

cd /temp/awstats-5.6
mkdir data
chmod 777 data

cd /tmp/awstats-5.6/wwwroot/cgi-bin/
cp awstats.model.conf awstats.conf


我diff了一下，修改以下几行：
LogFile="/etc/httpd/logs/access_log"　<--我的log在这儿，你的自己找啊。
SiteDomain="你的域名"
DirData="/tmp/awstats-5.6/wwwroot/data"
DirCgi="/awstat/cgi-bin"
DirIcons="/awstat/icon"
MiscTrackerUrl="/awstat/js/awstats_misc_tracker.js"
Lang="cn"
ShowFlagLinks="cn en fr de nl es"
LoadPlugin="tooltips"
LoadPlugin="geoip"
LoadPlugin="urlalias"
LoadPlugin="hashfiles"
LoadPlugin="timehires"
LoadPlugin="timezone +8"此处不用，这个plugin是针对IIS的

　　
　　4、安装Plugin，很费劲，以下是我的安装过程，
　　安装GeoIPfree 时需要先安装Geo::IPfree version 0.2+ (from Graciliano M.P.) ，而后者需要Memoize.pm;最后运行时发现，不能找到LookUp.al错误，来不及hack了，只能放弃这个插件。
　　安装GeoIP 时需要先安装 Geo::IP or Geo::IP::PurePerl (from Maxmind) ，其中要要先装C版的GeoIP包。（此时发现服务器的不能连接域名服务器了，后来发现是由于防火啬设定的问题，我的IP不能访问DNS服务器，后来用了个sina和网上一个朋友提供的）。

　　5、初始化数据分析结果，生成index.html文件。

cd /tmp/awstats-5.6/wwwroot/cgi-bin/
perl awstats.pl -update

su -
echo "/tmp/awstats-5.6/wwwroot/cgi-bin/awstats.pl -config=awstats.conf -update -output > /tmp/awstats-5.6/wwwroot/index.html" > /etc/cron.daily/awstats.cron
chmod 755 /etc/cron.daily/awstats.cron
exit

cd plugins
perl ../../../tools/urlaliasbuilder.pl -urllistfile=../../data/awstats072003.txt -site=你的域名
/etc/cron.daily/awstats.cron

　　6、如果一切正常，在客户端通过ie访问http://你的域名/awstat/就可以看到结果了。

　　7、车东的文章里有一些hack，可以加上啊。
　　参考他的文章，由于我已经加了timezone插件，所以他对时区的hack就不用做了。根据他的提示，增加了163和3721两个中文搜索引擎。但是，对google的UTF－8的解码问题仍没有解决，表现为显示页面上有乱码。但却有一行不乱，怪哉！

意外收获：　安装了N多perl包，学会perl程序的一般安装过程：

perl Makefile.pl
make
make test
su -
make install
