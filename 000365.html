学习struts笔记（1）

　　去年，跟着《程序员》学过一阵，今年再学一阵。国内对struts的研究和学习已经大有人在，更有一些人用struts做了一些实用开发，以致发布学习笔记前很是犹豫，恐见笑大方。这次学习，主要是在无意中发现，struts1.1有很多新的内容，原来知道的和现在网上常见大都是据以前的版本的写成的，因而便从struts网站开始学，不曾想学来学去，“学回家”了。：）

Welcome to Struts!


　　这个项目的目标，是提供一个创建Java Web 应用程序的开源框架。Struts 框架的核心是一个灵活的控制层，它基于一些标准技术，如Java Servlets, JavaBeans, ResourceBundles, and XML, 以及种的Jakarta Commons 程序包. Struts 鼓励应用程序采用基于Model2的架构，这是经典的Model-View-Controller (MVC)设计模式的一个变体。

　　Struts提供了自己的Controller组件，并集成其它技术以提供Model 和view。对于Model，Struts可以与标准的数据访问技术交互,如 JDBC 和EJB, 以及大多数第三方程序包，如 Hibernate, iBATIS, 或 Object Relational Bridge. 对于View, Struts 与JSP(JavaServer Pages), 包括JSTL和JSF, 以及 Velocity Templates, XSLT, 和其它表示系统一起工作得很好。.

　　Struts框架可为每个专业web应用系统提供赖以生存的支撑。基于公开发布的标准和经过验证的设计模式，Struts 帮助你为你的应用开发创建一个可扩展的开发环境。

　　Struts 是 Apache Jakarta 项目的一部分，受Apache软件基金赞助.

struts简介


　　一个web应用，利用发布描述符（ deployment descriptor）来初始化如servlet和taglibs等资源。发布描述符是一个XML文档，且被名为 "web.xml". 同样地， Struts 利用一个配置文件（名为struts-config.xml）来初始化它自己的资源。这些资源包收集用户输入的ActionForms，将用户输入指引到服务器端的Actions的 ActionMappings ，以及用以选择输出页面的ActionForwards 。

这儿有一个用以完成登录任务的简单structs应用的配置文件：

<?xml version="1.0" encoding="ISO-8859-1" ?>
<!DOCTYPE struts-config PUBLIC
"-//Apache Software Foundation//DTD Struts Configuration 1.2//EN"
"http://jakarta.apache.org/struts/dtds/struts-config_1_2.dtd">
<struts-config>
<form-beans>
<form-bean
name="logonForm"
type="app.LogonForm"/>
</form-beans>
<action-mappings>
<action
path="/Welcome"
forward="/pages/Welcome.jsp"/>
<action
path="/Logon"
forward="/pages/Logon.jsp"/>
<action
path="/LogonSubmit"
type="app.LogonAction"
name="logonForm"
scope="request"
validate="true"
input="/Logon.do">
<forward
name="success"
path="/Welcome.do"/>
<forward
name="failure"
path="/Logon.do"/>
</action>
<action
path="/Logoff"
type="app.LogoffAction">
<forward
name="success"
path="/Welcome.do"/>
</action>
</action-mappings>
<message-resources parameter="resources.application"/>
</struts-config>

　　在struts配置文件中，还可以指定其它一些资源。你可以借助 Struts Validator来为ActionForms提供验证，。利用Tiles, 可以帮助你创建页面。

　　Struts 是一个可扩展的框架。Struts发布的每一个类，都可以用你自己的类来替代。你的类的属性值，可以用Digester（解析xml以配置bean的程序）的set-property 功能来设置. 这是如此多的人为struct提供扩展功能的原因之一。Struts提供了基本的框架，但你可以自己的方式开发应用。

　　要得到更多的关于structs的信息，请参考用户指南（ User Guide ）和开发指南（（ Developer Guides）.

　　Struts是一个志愿者项目，所有支持都是由志愿者无偿提供的。这些文档以及邮件列表，是学习使用struts的基本途径。接下来的几个页面，将尽力助你理解，有哪些资源可以用。由于Struts 是一个志愿者项目，我们的资源有限，因此我们将首先帮助你学会自助。：）

Struts 是任何项目的最佳选择吗？


　　答案是不。
　　如果你需要写一个很简单的程序，只有几个页面，你大可以采用只使用jsp的Model 1方案。但是，如果你要写一个复杂一点的程序，有成十上百的页面，那将需要很多的时间来管理，这时用struts会用所帮助。要想知道 Model 1或 MVC/Model 2 哪一个适合你，请参考 Understanding JavaServer Pages Model 2 architecture and Issues in Struts Adoption.
