让人迷失的wiki

　　内容也是爆多，在收藏夹中建了个WIKI目录，以后慢慢看吧。

　　1：30了，有点晚了。

　　今天下班后，去了wikiwikiweb的老家一趟，结果发现有非常多的wiki程序，一时真搞不清哪个好哪个坏了。后来找到RandyKramer推荐了TWiki,又见台湾一个blogger也说其功能非常强大，就试它了。：）

　　到http://twiki.org找到下载连接，填写注册单后，会收到寄来的下载地址。下载文件有两种格式，TWiki20030201.zip或TWiki20030201.tar.gz，下面以后者为例说明。

　　1、将TWiki20030201.tar.gz上传到服务器。
　　2、我要在/usr目录下安装，所以
cd /usr
mkdir wiki
cp ~/TWiki20030201.tar.gz .
tar zxvf TWiki20030201.tar.gz
　　3、修改apache的设置文件，增加以下几行：
################My Wiki powered by TWiki#####################
 ScriptAlias /wiki/bin/ "/usr/wiki/bin/"
 Alias /wiki/ "/usr/wiki/"
 <Directory "/usr/wiki/bin">
    Options +ExecCGI
    SetHandler cgi-script
    Allow from all
 </Directory>
 <Directory "/usr/wiki/pub">
    Options FollowSymLinks +Includes
    AllowOverride None
    Allow from all
 </Directory>
 <Directory "/usr/wiki/data">
    deny from all
 </Directory>
 <Directory "/usr/wiki/templates">
    deny from all
 </Directory>
################My Wiki powered by TWiki#####################
　　4、修改TWiki.cfg文件
cd /usr/wiki/lib
vi TWiki.cfg
将以下几进行修改：
$defaultUrlHost = "http://hedong.3322.org";<------用你自己的域名
# %SCRIPTURLPATH% : cgi-bin URL path for TWiki:
$scriptUrlPath = "/wiki/bin";
# %PUBURLPATH% : Public data URL path for TWiki (root of attachments) :
$pubUrlPath = "/wiki/pub";
# Public data directory (file path not URL), must match $pubUrlPath :
$pubDir = "/usr/wiki/pub";
# Template directory :
$templateDir = "/usr/wiki/templates";
# Data (topic files) root directory (file path not URL):
$dataDir = "/usr/wiki/data";

$useLocale = 1;
$siteLocale = "zh_CN.UTF-8";
　　5、重启apache
service httpd restart
　　6、客户端测试
　　访问 http://hedong.3322.org/wiki/，点击get started，如果出现twiki网站的页面，OK！

　　需要说明的是，显示和编辑的页面都在$TWIKIHOME/data/下，要置可写属性哟！
